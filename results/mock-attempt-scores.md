## Where the length guard should live

Both places. Client-side (Next.js form) for instant feedback with the same 40/300-word limits and messages, AND inside `debrief-grade-attempt` before the model call — the API is the trust boundary; client validation is a UX nicety that anyone with curl can skip. The Lambda check is the enforcing copy. Reject over-long attempts, never truncate (truncation grades text the user didn't intend as their complete argument).
